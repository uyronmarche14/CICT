# CICT Website Roadmap

## Version 0: Foundation Prototype

### Goal
Prove the main public website and admin flow.

### Features

- Basic homepage
- Admin login
- News management
- Event management
- Organization data
- Member data

### Success Criteria

- Admin can access the panel
- Public can browse content
- Data is stored correctly

---

## Version 1: Core Public Website MVP

### Goal
Launch a working department website with admin-managed public content.

### Features

- Responsive public website
- News list and detail
- Event list and detail
- Organization pages
- Member pages
- Admin dashboard
- User management
- Role-based route protection
- Image upload

### Success Criteria

- Department can publish real content
- Public can browse core pages
- Website feels usable and modern

---

## Version 2: Content Management Improvement Version

### Goal
Improve admin usability and completeness of content modules.

### Features

- Better editing flows
- Richer organization management
- Richer member management
- Improved announcement presentation
- More complete page content

### Success Criteria

- admins can update content more easily
- more sections are no longer placeholders

---

## Version 3: Full Department Information Version

### Goal
Complete the missing public informational pages.

### Features

- Full About page
- Full Academics page
- Full Admissions page
- Full Student Life page
- Stronger Contact page

### Success Criteria

- the website feels complete as a department information platform

---

## Version 4: Governance and Operations Version

### Goal
Improve system control and accountability.

### Features

- Audit log views
- Better permission management UI
- Stronger admin activity review
- More complete content workflow tracking

### Success Criteria

- admins can review system activity
- access control becomes easier to manage

---

## Version 5: Advanced Experience Version

### Goal
Strengthen presentation and platform maturity.

### Features

- richer profile content
- richer organization pages
- better analytics or content insights
- stronger workflow consistency
- improved public polish

### Success Criteria

- platform feels complete, polished, and sustainable

---

# Recommended Development Order

```text
V0 Foundation
→ V1 Core Public Website MVP
→ V2 Content Management Improvements
→ V3 Full Department Information
→ V4 Governance and Operations
→ V5 Advanced Experience
```

# Core Strategy

Build the public and admin foundation first. Then improve completeness, workflow quality, and long-term maintainability.
