# CICT Website Product Requirements Document

## 1. Product Overview

### Product Name
**CICT Website and Admin Content Management Platform**

### Product Type
Department website, information hub, content publishing system, event platform, and organization showcase system.

### Product Description
The CICT Website is a digital platform for the College of Information and Communication Technology. It combines a public-facing website with an internal admin panel so authorized users can manage content without editing code directly.

The platform supports:

- public information pages,
- official news,
- announcements,
- events,
- organization profiles,
- member profiles,
- admin user management,
- and role-based content administration.

### Product Vision
To become the central digital platform of CICT for communication, visibility, organization showcasing, and department-managed public information.

### Product Mission
To make CICT information more organized, accessible, modern, and maintainable for students, visitors, faculty, and administrators.

## 2. Problem Statement

Many department websites are static, difficult to update, visually outdated, or dependent on developers for even simple content changes.

This creates common problems:

- important announcements are delayed,
- event information is hard to maintain,
- organizations are not presented clearly,
- member details become outdated,
- news and achievements are not consistently published,
- and content management becomes manual and inefficient.

CICT needs a system where authorized admins can manage content directly while the public receives a cleaner and more modern website experience.

---

## 3. Goals and Objectives

### Primary Goals

1. Create a modern public-facing CICT website.
2. Allow admins to manage content through a secure admin panel.
3. Publish and organize official news, announcements, and events.
4. Showcase organizations and members in a professional way.
5. Reduce dependency on manual code edits for content updates.
6. Build a foundation for future academic and student-information expansion.

### Operational Goals

1. Make updates faster for the department.
2. Improve visibility of department achievements and activities.
3. Keep organization and member information centralized.
4. Make public information easier to browse on desktop and mobile.

## 7. MVP Definition

The MVP is a working department website and admin panel where authorized users can:

- login,
- manage users,
- manage news,
- manage announcements,
- manage events,
- manage organizations,
- manage organization members,
- upload images,
- and publish content to the public website.

The public can:

- browse the homepage,
- read news,
- view events,
- view organizations,
- and open member profiles.

---

## 8. Key Features

### Public Website
A responsive public website that displays CICT information and content.

### News Publishing
Admins can create and maintain official news posts.

### Announcement Management
Admins can post reminders, notices, and urgent information.

### Event Management
Admins can create and update events while users can browse event details.

### Organization Showcase
Organizations can be presented with identity, mission, and achievements.

### Member Showcase
Members and officers can be presented with profile details and social links.

### Admin Management
Authorized users can manage content without direct code editing.

### Role-Based Access
Different admin roles can have different levels of control.
